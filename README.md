# pars-openstreetmap

Парсер инфраструктурных объектов с открытых карт (OpenStreetMap): школы,
детские сады, коммерческие точки (кофейни, спортзалы, аптеки и т.д.).

Модель: задаётся **центральная точка** (координатами или названием места) и
**радиус**, после чего анализируется вся инфраструктура в этом круге.

Источник данных — **OpenStreetMap** через бесплатный **Overpass API**,
геокодинг названий мест — через **Nominatim** (тоже OSM).
Архитектура расширяемая: чтобы добавить 2GIS/Yandex/Google Places, достаточно
реализовать интерфейс `DataSource` (см. `osm_parser/sources/base.py`).

## Возможности

- Парсинг объектов по категориям и группам в радиусе вокруг точки:
  образование, питание, спорт, здоровье, торговля, **транспорт** (метро, вокзалы),
  **общественные пространства** (парки, площадки, стадионы), **природа** (реки,
  водоёмы, леса), **промышленность** (заводы, промзоны).
- **История поиска** в БД: каждый поиск запоминается; записи можно
  **переименовывать** (`--rename`) и **повторять** (`--rerun`).
- Центр задаётся координатами (`--coords`) или названием/адресом (`--place`).
- Радиус — произвольный (`--radius`); готовые пресеты: **1, 2, 5, 10 км**.
- Сохранение в **SQLite** (upsert по идентификатору объекта).
- Экспорт в **GeoJSON** (для карт/ГИС) и **CSV**.
- **Интерактивная веб-карта** (Leaflet): круг поиска, маркеры с цветом по
  группам, попапы с адресом/телефоном/часами, фильтр слоёв и легенда.
  Самодостаточный HTML — открывается двойным кликом, сервер не нужен.
  На карте есть кнопка **«⬇ Скачать .md»** — по клику формирует и сохраняет
  отчёт по всем объектам в **Markdown** (сводная таблица + детализация по
  категориям). Выгрузка происходит только по нажатию, не автоматически.
- Аналитика: количество по категориям/группам, **плотность** (объектов на км²),
  распределение по **концентрическим кольцам** удалённости от центра.

## Установка

```powershell
py -m venv .venv
.venv\Scripts\python.exe -m pip install -r requirements.txt
```

## Развёртывание (публичный доступ)

Приложение готово к хостингу: сервер слушает `0.0.0.0`, порт берёт из `$PORT`,
путь к БД настраивается через `DB_PATH`. В репозитории есть `Procfile`
(для Render/Railway) и `Dockerfile` (для Fly.io и любого контейнерного хостинга).

Команда запуска в проде:

```
uvicorn server:app --host 0.0.0.0 --port $PORT
```

Варианты хостинга для небольшой аудитории:
- **Render** (проще всего): бесплатный Web Service из GitHub-репозитория, выдаёт
  ссылку вида `https://<имя>.onrender.com`. На бесплатном тарифе сервис «засыпает»
  после простоя (первый заход ~30–60 c) и файловая система эфемерна — история
  поиска сбрасывается при перезапуске. Для постоянной истории — Persistent Disk и
  переменная `DB_PATH=/data/osm_poi.db`.
- **Fly.io**: контейнер из `Dockerfile` + том (volume) для постоянной SQLite-истории.
- **Cloudflare Tunnel / ngrok**: мгновенная ссылка на локально запущенный сервер,
  без деплоя (но твой компьютер должен быть включён).

> ⚠️ Сервис обращается к публичным Overpass и Nominatim, у которых есть лимиты и
> правила использования. Для небольшого числа пользователей это нормально; при
> росте нагрузки стоит поднять свой инстанс Overpass и кэшировать запросы.

## Веб-приложение (интерактивный режим)

Запусти сервер и открой страницу в браузере:

```powershell
.venv\Scripts\python.exe server.py
# затем открыть http://127.0.0.1:8000
```

Режимы поиска (переключатель сверху):
- **⭕ Окружность** — поставь точку кликом по карте (или найди место по названию),
  выбери радиус. Показывает маркеры, плотность и кольца удалённости.
- **▭ Прямоугольник** — нажми «Рисовать» и обведи прямоугольную область на карте.
- **✏️ Фигура** — нарисуй произвольный полигон (клики по вершинам).
- **🏙 Город** — введи название города/района. Поиск идёт строго внутри его
  **административной границы** (реальный полигон из OSM), контур рисуется на карте.

Рисование фигур — через плагин Leaflet-Geoman; поиск по нарисованной области
выполняется фильтром Overpass `poly:`. Нарисованные фигуры сохраняются в истории
и восстанавливаются при повторе (↻).

Маркер центра можно перетаскивать. История поисков — справа внизу, с
кнопками повторить (↻), переименовать (✎) и удалить (🗑); повтор восстанавливает
и режим (радиус/город/фигура).

Кнопка **«⬇ Скачать данные»** спрашивает формат выгрузки и сохраняет текущий
результат в одном из трёх форматов:
- **Markdown** — сгруппировано по категориям, с координатами объектов;
- **Excel (.xlsx)** — плоская таблица с автофильтром (генерируется на сервере);
- **CSV** — плоская таблица (с BOM для корректной кириллицы в Excel).

> ⚠️ Поиск по крупному городу с категориями вроде «леса/реки/все кафе» может
> быть медленным и тяжёлым для публичного Overpass — ограничивай набор категорий.

Ниже описан альтернативный режим — **CLI** (для пакетной работы и экспорта).

## Использование

```powershell
# Список доступных категорий
.venv\Scripts\python.exe main.py --list-categories

# Вся инфраструктура в радиусе 2 км от координаты → в БД
.venv\Scripts\python.exe main.py --coords 55.751,37.618 --radius 2

# Школы и детсады в радиусе 1 км от места (по названию)
.venv\Scripts\python.exe main.py --place "Казань, Кремль" --radius 1 -c school kindergarten

# Коммерция в радиусе 5 км + интерактивная карта + анализ по 5 кольцам
.venv\Scripts\python.exe main.py --place "Москва, Арбат" --radius 5 -c food sport retail `
    --map infra.html --rings 5
```

Карту `infra.html` затем открывают двойным кликом в браузере.

### История поиска

```powershell
# Показать историю
.venv\Scripts\python.exe main.py --history

# Переименовать запись #3
.venv\Scripts\python.exe main.py --rename 3 "Центр Казани — вся инфраструктура"

# Повторить поиск из истории
.venv\Scripts\python.exe main.py --rerun 3

# Удалить запись истории
.venv\Scripts\python.exe main.py --delete 3

# Задать имя записи сразу при поиске
.venv\Scripts\python.exe main.py --coords 55.75,37.62 -r 2 --name "Тверская, 2 км"
```

История хранится в той же SQLite БД (`--db`), в таблице `searches`.

Категории можно указывать поштучно (`school`), группами (`food`) или `all`.
Радиус — любой в км; пресеты 1/2/5/10 — просто удобные значения.

### Где взять координаты

Координаты точки можно скопировать из любой карты (в Яндекс/Google по
правому клику), либо просто указать название места через `--place`.

## Структура

```
osm_parser/
├── categories.py        # таксономия: категории ↔ OSM-теги (расширяй здесь)
├── models.py            # модель POI (единая для всех источников)
├── geo.py               # геометрия (расстояния haversine)
├── geocode.py           # геокодинг названий мест через Nominatim
├── storage.py           # SQLite + экспорт GeoJSON/CSV
├── webmap.py            # генерация интерактивной карты Leaflet (HTML)
├── analytics.py         # статистика, плотность, кольца удалённости
└── sources/
    ├── base.py          # абстрактный DataSource + Circle (центр+радиус)
    └── overpass.py      # реализация для OSM Overpass API
main.py                  # CLI
```

## Заметки по данным

- В крупных городах РФ покрытие OSM хорошее; в малых городах коммерческие
  объекты (часы работы, телефоны) могут быть неполными.
- Публичный Overpass API просит соблюдать вежливый rate limit — клиент делает
  паузы и повторы при `429/504`. Для больших объёмов имеет смысл поднять свой
  инстанс Overpass или работать с дампами Geofabrik.
